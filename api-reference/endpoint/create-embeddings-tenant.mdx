---
title: "Create Embeddings Tenant"
description: "Prepare a tenant for direct embedding uploads. Use POST /tenants/create with is_embeddings_tenant: true."
---

<Warning>
  The standalone `POST /embeddings/create_tenant` endpoint has been removed. Use `POST /tenants/create` with `"is_embeddings_tenant": true` instead.
</Warning>

To create a tenant dedicated to directly storing and querying embeddings, call `POST /tenants/create` and set `is_embeddings_tenant` to `true`. You can optionally specify `embeddings_dimension` (defaults to `1536`).

### Examples

<Tabs>
  <Tab title="API Request">
    ```bash
    curl -X 'POST' \
    'https://api.hydradb.com/tenants/create' \
    -H 'accept: application/json' \
    -H 'Content-Type: application/json' \
    -H 'Authorization: Bearer {API_KEY}' \
    -d '{
      "tenant_id": "embeddings_tenant_123",
      "is_embeddings_tenant": true,
      "embeddings_dimension": 1536
    }'
    ```
  </Tab>
  <Tab title="TypeScript">
    ```ts
    const collection = await client.tenant.create({
      tenant_id: "embeddings_tenant_123",
      is_embeddings_tenant: true,
      embeddings_dimension: 1536,
    });
    ```
  </Tab>
  <Tab title="Python (Sync)">
    ```python
    # Async usage is similar, just use async_client and await
    collection = client.tenant.create(
        tenant_id="embeddings_tenant_123",
        is_embeddings_tenant=True,
        embeddings_dimension=1536,
    )
    ```
  </Tab>
</Tabs>

### Request Body Fields

| Field                 | Type    | Required | Description                                             |
| --------------------- | ------- | -------- | ------------------------------------------------------- |
| `tenant_id`           | string  | No       | Desired tenant ID. Auto-generated if omitted.           |
| `is_embeddings_tenant`| boolean | Yes      | Set to `true` to configure the tenant for raw embeddings |
| `embeddings_dimension`| integer | No       | Vector dimension (default: `1536`)                      |

## Sample Response

```json
{
  "status": "success",
  "message": "Tenant 'embeddings_tenant_123' created successfully.",
  "tenant_id": "embeddings_tenant_123"
}
```

> Use this only if you plan to upload embeddings directly (via `/embeddings/insert_raw_embeddings`) and query them (via `/embeddings/*`). For standard document, text, or app-sources ingestion, use `POST /tenants/create` without `is_embeddings_tenant`.
