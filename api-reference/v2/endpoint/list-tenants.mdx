---
title: "List Databases"
description: "List all databases created. "
openapi: "api-reference/v2/openapi.json GET /databases"
---

The response separates active or provisioning databases (in `data.databases`) from databases whose provisioning failed (in `data.failed_databases`). Use [Database Status](https://docs.hydradb.com/api-reference/v2/endpoint/tenant-status) to confirm readiness before ingestion.

<RequestExample>

```python Python SDK
response = client.databases.list()
```

```typescript TypeScript SDK
const response = await client.databases.list();
```

```bash cURL
curl -X GET 'https://api.hydradb.com/databases' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2"
```

</RequestExample>

## Retry notes

- If provisioning failed for a database, `data.failed_databases` contains diagnostic entries as shown in the **Provisioning issue** tab.
- **Retry failed databases:** If a database appears in `data.failed_databases`, re-create that database with `POST /databases` after addressing the reported issue. Poll status again before ingestion.
